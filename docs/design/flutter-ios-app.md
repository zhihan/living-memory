# Flutter iOS App — Plan

## Goal

Build a native iPhone app using Flutter that reuses the existing FastAPI backend.
The app covers the same flows as the web app, optimized for mobile use.

## Authentication

- **Google Sign-In** via the `google_sign_in` Flutter package
- After Google auth, exchange the Google credential for a **Firebase ID token** using the `firebase_auth` package (`signInWithCredential`)
- All API calls attach the Firebase ID token as `Authorization: Bearer <token>`, identical to the web app
- No backend changes needed — the existing `_verify_firebase_token` endpoint works as-is
- Firebase project: `living-memories-488001` (same config as web)

## API Layer

The app talks to the same deployed API (`https://living-memories-488001.web.app` or configurable base URL). All existing endpoints are reused:

| Area | Endpoints |
|------|-----------|
| Auth | Firebase ID token (client-side only) |
| Workspaces | `POST/GET/PATCH/DELETE /v2/workspaces/...` |
| Members & Invites | `GET/POST/DELETE /v2/workspaces/{id}/members`, `/invites/{id}/accept` |
| Series | `POST/GET/PATCH/DELETE /v2/series/...`, `/check-in-report` |
| Occurrences | `POST/GET/PATCH /v2/occurrences/...`, `/generate` |
| Check-ins | `POST/GET/PATCH/DELETE /v2/occurrences/{id}/check-ins`, `/my-check-in` |
| Notifications | `GET/POST/DELETE /v2/.../notification-rules` |

No new endpoints are required.

## Screens

### 1. Sign-In
- Google Sign-In button
- Auto-navigate to Dashboard on success

### 2. Dashboard (Home)
- List of user's workspaces (from `GET /v2/workspaces`)
- "Create workspace" action
- Tap workspace → Workspace View

### 3. Workspace View
- Workspace title (tap to edit for organizers)
- List of series with schedule summary
- "New Series" form (title, frequency, weekdays, time, duration, location, check-in days)
- Members section with invite link generation
- Tap series → Series View

### 4. Series View
- Series details (title, schedule, description)
- Edit form (organizer/teacher)
- Next / last meeting cards
- Upcoming occurrences list with inline location editing
- Check-in report button → opens report table (participants × dates grid)
- Report window selector (last 5/10/20/50)

### 5. Occurrence View
- Date, title, location, online link, notes
- Status controls (organizer: scheduled/cancelled/completed)
- Enable check-in toggle (organizer)
- Edit occurrence overrides (organizer)
- Practice check-in button ("Mark as done" / "Done ✓ Undo")
- Check-in list (organizer/teacher: all check-ins; participant: own only)

### 6. Accept Invite
- Deep link handler for `/invites/{id}`
- Shows workspace name, calls `POST /invites/{id}/accept`

## Setup Steps

1. `flutter create --platforms ios event_ledger_app`
2. `flutterfire configure` with project `living-memories-488001`
3. Add Google Sign-In iOS config (GoogleService-Info.plist, URL schemes)
4. Implement auth_service → api_service → models → screens in order

## Deep Links

- Register URL scheme for invite links: `https://living-memories-488001.web.app/invites/{id}`
- Use Firebase Dynamic Links or Universal Links to route to AcceptInviteScreen

## Scope & Non-goals

**In scope (v1):**
- All screens listed above, matching web app functionality
- Google Sign-In authentication
- Check-in report table
- Pull-to-refresh on list screens

**Not in scope (v1):**
- Push notifications (can add later via FCM)
- Offline mode / local caching
- Android build (Flutter supports it, but focus on iOS first)
- Assistant / chat feature
- Telegram webhook management
- ICS export (can open in browser)

## Migration Path

The Flutter app and the web app coexist — both use the same backend and Firebase project. Users can switch between them freely. No data migration needed.
